CHAPTER 1: INTRODUCTION

1.1. Client Identification / Need Identification / Identification of Relevant Contemporary Issue
-----------------------------------------------------------------------------------------
The primary clients for this project are students, freshers, and professional job seekers navigating an increasingly automated recruitment landscape. 

CONTEMPORARY ISSUE: 
The professional world suffers from "Career Data Fragmentation." A candidate's identity is scattered across LinkedIn, local resume files, portfolio sites, and university ERPs. 

JUSTIFICATION:
- Statistics & Documentation: According to recent recruitment reports, nearly 75% of resumes are filtered out by Applicant Tracking Systems (ATS) before reaching a human recruiter. This is often due to poor keyword alignment and formatting errors.
- Consultation Problem: Job seekers face "Application Fatigue." Manually filling out 16+ sections of career data (Experience, Education, Projects, Volunteer, etc.) for every individual company portal is a significant barrier to productivity.
- Survey/Need: Preliminary surveys among university seniors indicate that 85% of students find current platforms (like LinkedIn) insufficient for "Specific Role Tailoring," and 90% desire a central "Single Source of Truth" that can automatically map data to various application formats.
- Agency Documentation: Reports from agencies like the World Economic Forum highlight the "skills gap" and the need for more granular, AI-validated career profiles to improve labor market matching.

1.2. Identification of Problem
------------------------------
The core problem is the lack of a centralized, interoperable system for managing professional identity. Current processes are manual, repetitive, and lack the intelligence to bridge the gap between a candidate's repository of skills and the specific, differing requirements of diverse global Job Descriptions (JDs). This results in missed opportunities for candidates and prolonged vacancy periods for employers.

1.3. Identification of Tasks
--------------------------
To resolve the identified problem, the following tasks have been defined:
- TASK 1: Literature Survey: Analyze existing recruitment technologies, ATS algorithms, and LLM-based parsing capabilities.
- TASK 2: Database Modeling: Design a flexible "Career ERP" schema that handles 16+ dynamic career sections.
- TASK 3: UI/UX Development: Build a high-fidelity frontend that allows for "Single-Click" data ingestion and manual verification.
- TASK 4: AI Extraction Engine: Implement Gemini-based parsers to translate messy PDFs (LinkedIn/Resumes) into structured JSON.
- TASK 5: Automation Services: Develop Playwright-based scripts to simulate human-in-the-loop job applications.
- TASK 6: Testing & Validation: Conduct efficiency tests comparing manual application speed vs. AI-assisted speed.

1.4. Technical Stack & Architecture
----------------------------------
The system is built using a modern, scalable, and secure "Enterprise-Ready" stack:
- Backend Framework: FastAPI (Python 3.10+) with Pydantic v2 for robust data validation and serialization.
- Frontend Core: React 19 (TypeScript) using Vite for high-speed HMR and optimized production builds.
- Database & ORM: PostgreSQL with SQLAlchemy 2.0 (Eager loading with joinedload) and Alembic for schema migrations.
- AI & LLM Processing: Google Gemini 1.5 Flash (via httpx) for context-aware extraction and skill mapping.
- Authentication & Security: python-jose (JWT), passlib (Bcrypt hashing), and Google OAuth2 integration.
- Document Engineering: PyPDF (Extraction of raw text from LinkedIn PDFs and resumes).
- Network & API: Axios (Frontend HTTP client) and httpx (Backend asynchronous client).
- Email Delivery: smtplib with secure App Passwords (TLS encryption) for transaction emails.
- Global Styling: Vanilla CSS with modern layout techniques (Flexbox, Grid) for a bespoke design system.
- Automation Engine: Playwright (for future-ready human-in-the-loop browser interactions).

1.5. Current Implementation Status (Milestones 1-3)
-------------------------------------------------
As of now, the following pillars of the Career ERP have been successfully implemented:
- Secure Identity Layer: Google OAuth and email-based OTP/Password Reset flows are fully operational.
- Triple-Injetion Pipeline: Users can build their identity via Manual Entry, LinkedIn PDF Import, or Resume AI Parsing.
- AI Metadata Extraction: Accurate extraction of 16+ career sections from unstructured PDF sources into a single schema.
- Data Transformation: Manual review system to verify AI-extracted data before it is persisted in the permanent ERP.
- User Dashboard: Real-time calculation of profile completion and professional identity summary.

1.6. Future Core Feature Roadmap (Milestones 4+)
-----------------------------------------------
The next phases of the project will focus on the 'Execution' layer of the platform:
- Automatic Resume Tailoring: Generating role-specific PDF resumes by selecting only relevant profile segments via AI.
- Explainable Relevance Scoring: Analyzing a Job Description against the ERP data to provide a "Confidence Score" and identify skill gaps.
- Multi-Tab Automation: Using Playwright to auto-fill job portals (Workday, Greenhouse, Lever) based on the ERP Identity.
- Skill Growth Analytics: Providing long-term insights into which skills the user should acquire to increase their success rates.
